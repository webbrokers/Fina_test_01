## Корневая причина проблем

**DOM 15769 элементов** - в 10 раз больше нормы!

125 карточек МФО × 4 таба × ~30 элементов = катастрофа. Браузер тратит секунды на парсинг и рендеринг скрытого контента, который пользователь никогда не увидит без скролла.

## 🔴 P1 - Critical (делать обязательно)

### 1.1 Отложенный рендеринг карточек МФО
**Где:** блок с карточками  
**Что делать:**
- **30 карточек остаются в HTML** — видны поисковым роботам, индексируются
- Остальные карточки — скрыты `display:none` в HTML (индексируются!) + кнопка “Показать ещё” показывает их через JS
- Из карточек, скрытых display:none, убрать тяжёлые ресурсы (картинки заменить на data-src)

**Пример:**
```html
<!-- 30 первых карточек — обычные, видные Google -->
<div class="loan-card">...</div>
...
<!-- Остальные — скрытые, но в HTML для индексации -->
<div class="loan-card hidden-card" style="display:none">...</div>
...
<button onclick="showMoreCards()"> Показать ещё </button>
<script>
function showMoreCards() {
  document.querySelectorAll('.hidden-card')
    .forEach((el, i) => { if(i < 20) el.style.display = ''; });
}
</script>
```

**Жуков DOM нет** — все карточки в HTML, просто скрытые. Браузер не рендерит скрытые элементы, Google видит все.

**Эффект:** DOM с 15769 → ~4000 активных элементов в памяти, LCP -8-12 сек

## 🟡 P2 - SEO & Performance

### 2.1 Canonical URL
**Где:** `<head>`
**Сейчас:** `<link rel="canonical" href="index.html">`
**Надо:** убрать или поставить полный URL тестовой страницы

```html
<link rel="canonical" href="https://webbrokers.github.io/Fina_test_01/collection/zajmy-s-plohoj-kreditnoj-istoriej/">
```

### 2.2 Alt для изображений
**Где:** 196 изображений, из них ~15 без alt
**Что делать:** добавить осмысленные alt-тексты к логотипам МФО

```html
<!-- Было -->
<img src="zaymer-logo.png">
<!-- Стало -->
<img src="zaymer-logo.png" alt="Логотип Займер">
```

### 2.3 Width/height для изображений
**Где:** все `<img>` без явных размеров
**Что делать:** добавить атрибуты чтобы предотвратить layout shift

```html
<img src="logo.png" width="120" height="40" alt="...">
```

### 2.4 Lazy loading для изображений ниже fold
**Где:** изображения карточек МФО (все кроме первых 5-7)
**Что делать:**
```html

```
**Эффект:** -200-400 КиБ при начальной загрузке

## 🟢 P3 - Улучшения (nice to have)

### 3.1 Геотаргетинг - отложенная загрузка (НЕ удалять!)
**Где:** `cookie.js`, `wt-location.js` + список городов
**Почему оставляем:** геотаргетинг нужен на основном сайте, в тесте тоже оставляем
**Что делать:** скрипты загружать с `defer`. Список городов - загружать только по клику на меню городов (lazy fetch), НЕ при старте страницы
```html
<!-- Скрипты с defer -->
<script defer src="cookie.js"></script>
<script defer src="wt-location.js"></script>
```
Список городов подгружается через fetch() только когда пользователь кликает на кнопку выбора города.

### 3.2 Preload критического CSS
**Где:** `<head>`
```html
<link rel="preload" href="style.css" as="style">
```

### 3.3 Объединить инлайн-стили
**Где:** 10 блоков `<style>` в head
**Что делать:** объединить в один блок или вынести в файл

### 3.4 Убрать exit_popup.css если попап не нужен для теста
**Эффект:** -5 КиБ

### 3.5 Оптимизация адаптивных блоков (hidden-lg/hidden-xs)
**Контекст:** на странице 150 Bootstrap классов `hidden-lg`, `hidden-sm` и т..д. — блоки загружаются всегда, просто скрыты CSS.

**Что делать:**
- Всем картинкам внутри `hidden-lg`/`hidden-md` блоков — добавить `loading="lazy"`
- CSS стили для мобильных устройств пометить `media="(max-width:767px)"` — не блокирует рендеринг на десктопе

**Примечание:** Полное разделение на мобильную/десктопную версии — устаревший подход. Google Mobile-First работает с единым HTML + адаптивным CSS. Это правильная архитектура.

## ⚠️ Главный принцип оптимизации

> **Удалять ТОЛЬКО то, что точно не вызывается на этой странице И дублируется.**
> Всё остальное - переводить на `defer`/`async`, но не удалять.
> Геотаргетинг, слайдеры (кроме tiny), лайтбоксы, плагины - они нужны на основном сайте, в тесте тоже оставляем, просто загружаем правильно (отложенно).
